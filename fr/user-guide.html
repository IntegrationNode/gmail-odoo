Tout ce que vous devez savoir pour tirer le meilleur parti de l'intégration Odoo pour Gmail

Démarrage rapide (2 minutes, sans clé API)
  1. Installez l'extension et ouvrez Gmail.
  2. Connectez-vous à votre Odoo dans le même navigateur Chrome — l'extension utilise votre session active.
  3. Saisissez votre URL Odoo (ex. https://votreentreprise.odoo.com — HTTPS, sans barre oblique finale) et cliquez sur Connecter.
Ouvrez n'importe quel email et votre contact Odoo apparaît dans la barre latérale. C'est tout — pas de clé API, pas de code.

download Installation

Prérequis

  • Navigateur Chrome: Version 88 ou supérieure
  • Instance Odoo: Version 14.0 ou supérieure (on-premise, Odoo.sh ou Odoo Online plan Custom)

Installation étape par étape

  1. Visitez le Chrome Web Store
    Accédez à Intégration Odoo pour Gmail ou recherchez "Odoo Integration for Gmail" dans le Chrome Web Store.
  2. Ajouter à Chrome
    Cliquez sur le bouton "Ajouter à Chrome" et examinez les autorisations demandées.
  3. Vérifier l'installation
    Vous verrez un message de confirmation et l'icône de l'extension apparaîtra dans votre barre d'outils Chrome.
  4. Épingler l'extension (Recommandé)
    Cliquez sur l'icône puzzle dans la barre d'outils Chrome, trouvez "Odoo Integration for Gmail" et cliquez sur l'icône d'épingle.
Note concernant la "Navigation sécurisée renforcée" : Si vous voyez un message indiquant "Cette extension n'est pas approuvée par la navigation sécurisée renforcée", c'est normal pour les extensions nouvellement publiées. L'extension a été examinée et approuvée par le Chrome Web Store.

settings Configuration

Première configuration

  1. Ouvrir Gmail
    Accédez à mail.google.com et ouvrez n'importe quel email pour voir la barre latérale.
  2. Connectez-vous à Odoo
    Assurez-vous d'être connecté à votre instance Odoo dans le même navigateur — l'extension se connecte via votre session active.
  3. Entrer votre URL Odoo
    Entrez l'URL de votre instance Odoo (ex: https://votreentreprise.odoo.com). Utilisez HTTPS et n'incluez pas de barre oblique finale.
  4. Connecter
    Cliquez sur Connecter et l'extension vérifiera votre session active. Une clé API est facultative — nécessaire uniquement pour l'enregistrement automatique en arrière-plan, ou comme solution de secours si votre session expire.
info Ajouter une clé API est facultatif. Si vous en ajoutez une, les étapes de création varient selon les versions d'Odoo. Consultez le guide détaillé pour votre version →

Mise à jour de la configuration

Pour modifier votre URL Odoo, déconnectez-vous et reconnectez-vous :

  1. Cliquez sur le bouton menu (trois points verticaux ⋮) dans la barre latérale
  2. Sélectionnez Se déconnecter
  3. Entrez vos nouvelles informations

Pour ajouter ou mettre à jour votre clé API plus tard, ouvrez Paramètres → Compte → Connexion — sans avoir à vous déconnecter.

credit_card Gestion de l'abonnement

Comment s'abonner à PLUS

  1. Cliquez sur "Passer à PLUS" depuis le menu de la barre latérale (trois points dans l'en-tête)
  2. Complétez le processus de paiement sur Polar
  3. Recevez votre clé de licence par email

Activer votre Licence

Entrez votre clé de licence de l'une de ces façons :

  • Une fenêtre apparaîtra juste après le paiement - entrez votre licence ici
  • Allez dans Paramètres > Compte > Activer la Licence
  • Cliquez à nouveau sur "Passer à PLUS" et sélectionnez "J'ai déjà une licence"
Informations sur la Licence : Votre licence fonctionne sur jusqu'à 3 instances (différents ordinateurs ou navigateurs). Elle doit être utilisée avec le même compte Gmail que celui utilisé pour l'achat.
Essai Gratuit : PLUS inclut un essai gratuit de 14 jours. Vous ne serez pas facturé avant la fin de la période d'essai. Annulez à tout moment avant pour éviter les frais.
Tarification régionale : Nous offrons des prix justes. Les prix sont automatiquement ajustés lors du paiement via une réduction.

Comment annuler votre abonnement

Vous pouvez annuler votre abonnement PLUS à tout moment :

  1. Accédez au portail client Polar (lien dans votre email d'achat)
  2. Connectez-vous avec l'email utilisé pour votre abonnement
  3. Cliquez sur "Annuler l'abonnement" dans les détails de votre abonnement

Vos fonctionnalités PLUS resteront actives jusqu'à la fin de votre période de facturation en cours.

mail Enregistrement d'emails

Comment enregistrer un email

  1. Ouvrez l'email que vous souhaitez enregistrer
  2. Consultez le contact dans la barre latérale
  3. Trouvez l'enregistrement auquel vous voulez associer l'email (opportunité, tâche ou ticket)
  4. Cliquez sur l'icône enveloppe à côté de l'enregistrement
  5. Le bouton devient une coche verte une fois enregistré

Ce qui est enregistré

  • Sujet de l'email (dans l'en-tête)
  • Corps de l'email (formaté en HTML)
  • Informations expéditeur/destinataire
  • Date et heure
  • Toutes les pièces jointes (fichiers, images, PDFs)
  • Pied de page "Enregistré depuis Gmail Inbox"
Enregistré comme Note Interne : Les emails sont enregistrés comme notes internes dans le chatter d'Odoo. Cela signifie que seuls les utilisateurs internes (employés) peuvent les voir. Les utilisateurs du portail (clients) NE verront PAS les emails enregistrés. Les notifications ne sont envoyées qu'aux abonnés internes de l'enregistrement.

Détection des doublons

L'extension dispose de deux niveaux de détection des doublons :

  1. Indicateur visuel (0-6 heures): Vous aide à identifier d'un coup d'œil quels enregistrements contiennent déjà cet email
  2. Validation côté serveur (toujours): Odoo empêche les doublons même si l'indicateur visuel a expiré
Important : Même sans l'indicateur visuel, vous ne pouvez pas créer accidentellement de doublons. Le serveur Odoo valide chaque demande d'enregistrement et rejette les doublons.

sync Auto-logging Plus

Auto-logging links a Gmail thread to one or more Odoo records so that every new email in that thread is logged into the record's chatter automatically — no need to click the envelope icon for each reply.

Private Beta: Auto-logging is currently in private beta. It will be available to all PLUS subscribers soon. To request early access, contact support.

Requirements

  • PLUS plan (and beta access during private beta)
  • Gmail API connection (granted from Settings > Auto-logging > Connect)
  • Auto-logging toggle enabled in Settings > Auto-logging

How to Activate Auto-logging on a Thread

  1. Open the email thread in Gmail and view the contact in the sidebar
  2. Find the record you want to track (opportunity, task, ticket, or contact)
  3. Click the sync icon next to the record. The icon turns green when active and a confirmation toast appears
  4. From now on, every new email in this thread is logged automatically into the chatter of that record
Only future emails: Auto-logging is purely forward-looking. Emails that already existed in the thread before activation are NOT logged retroactively. To log older emails, use the manual envelope button as usual.

What Gets Logged

Each new email is added as an internal note in the record's chatter, with:

  • A Gmail-style header (From, To, Date, Subject)
  • The email body without quoted history
  • The original date of the email (not the time it was processed)
  • Linked partners — recipients are matched in Odoo by email and attached to the message
  • An "Auto-logged from Gmail" footer linking back to the extension
Logged as Internal Note: Auto-logged emails are saved as internal notes (mt_note), not public messages. Only internal Odoo users can see them — portal users (customers) won't.

Linking a Thread to Several Records

You can activate auto-logging for the same thread on multiple records (for example, an opportunity and a task related to the same conversation):

  • 1 thread → 1 record: standard case.
  • 1 thread → several records: every new email is logged on each linked record sequentially.
  • The same record cannot be linked twice from the same thread — duplicates are rejected.

Managing Active Auto-logs

From the sidebar (per thread)

When the open thread has any active mappings, an "Auto-logging (N)" section appears at the top of the sidebar:

  • Click the section to expand it and see every record linked to the current thread
  • Click on a record's name to open it in Odoo
  • Click the ✕ button to deactivate auto-logging for that specific record

From Settings (across all threads)

Open Settings > Auto-logging to manage every active mapping:

  • Enable or disable auto-logging globally with a single toggle
  • Connect or reconnect the Gmail API
  • See the list of active mappings, the count, and the time of the last successful check
  • Deactivate all mappings at once with the "Deactivate all" button

Limits & Edge Cases

  • Not retroactive: Only emails that arrive after activation are logged. Existing emails must be logged manually.
  • Auto-cleanup after 90 days: A mapping with no new email activity for 90 days is removed automatically. Activate it again to resume tracking.
  • Record deleted in Odoo: If the linked record is deleted in Odoo, the mapping is removed automatically the next time the extension tries to log into it.
  • Gmail disconnected: If the Gmail API connection expires or is revoked, auto-logging stops for all threads until you reconnect from Settings > Auto-logging.

groups Utiliser les contacts associés Plus

La section Contacts associés du panneau latéral liste les collègues de la même entreprise que l'expéditeur de l'e-mail. Vous pouvez y effectuer des actions rapides sur chaque contact sans quitter Gmail.

Ouvrir un contact dans Odoo

Cliquez n'importe où sur la carte du contact — l'avatar, le nom ou la fonction — pour ouvrir sa fiche dans Odoo dans un nouvel onglet.

Envoyer un nouvel e-mail à un contact

Cliquez sur l'icône d'e-mail à côté de l'adresse du contact. Si aucun brouillon n'est ouvert, Gmail ouvre une nouvelle fenêtre de rédaction avec cette adresse pré-remplie dans le champ À.

Ajouter un contact à un e-mail en cours de rédaction

Si une fenêtre de rédaction Gmail est déjà ouverte au moment où vous cliquez sur l'icône d'e-mail, le contact est ajouté comme destinataire de ce brouillon au lieu d'en ouvrir un nouveau. L'extension choisit où : si le champ À est vide, le contact y est placé ; s'il contient déjà quelqu'un, le contact est ajouté en CC et la fenêtre de rédaction est minimisée pour vous permettre de continuer à cliquer sur d'autres contacts et construire la liste des destinataires.

Copier l'e-mail ou le téléphone d'un contact

Cliquez sur le texte de l'e-mail ou du téléphone — pas sur l'icône à côté — pour le copier dans votre presse-papiers. Une petite notification confirme la copie.

Appeler un contact

Cliquez sur l'icône de téléphone pour démarrer un appel. L'action est transmise à l'application que votre système associe aux numéros de téléphone — le composeur de votre mobile, FaceTime, Skype, Microsoft Teams, etc. Si aucune application n'est configurée, le clic ne fait rien.

history Naviguer dans les activités récentes Plus

La section Activités récentes affiche les derniers messages et conversations issus de la fiche du contact dans Odoo, pour que vous voyiez ce qui a été dit sans quitter Gmail.

Voir exactement quand une activité s'est produite

Chaque entrée affiche une date relative comme « il y a 5 jours ». Survolez la date pour voir le jour et l'heure exacts, formatés selon la langue de votre navigateur.

Aller directement à un message dans Odoo

Cliquez sur une entrée des Activités récentes pour ouvrir la fiche Odoo associée dans un nouvel onglet. Sur Odoo 18 et versions ultérieures, le clic va plus loin : Odoo défile jusqu'à ce message précis et le met en surbrillance. Sur Odoo 16 et 17, la fiche s'ouvre normalement — le clic vous évite la navigation, mais il faudra défiler pour trouver le message.

notifications Organisation des Notifications Plus

Organisez les emails de notification Odoo directement dans votre boîte de réception Gmail. L'extension identifie les emails de notification d'Odoo et ajoute des badges visuels pour voir instantanément à quel module ils appartiennent et si vous avez été mentionné.

Prérequis : Activer les Notifications par Email dans Odoo

Pour que cette fonctionnalité fonctionne, votre utilisateur Odoo doit être configuré pour recevoir les notifications par email (et non dans la boîte Odoo) :

  1. Dans Odoo, allez dans votre Profil (menu en haut à droite > Mon Profil)
  2. Dans Préférences, trouvez le paramètre "Notification"
  3. Sélectionnez "Gérer par e-mails" (au lieu de "Gérer dans Odoo")
Cela garantit qu'Odoo envoie les emails de notification à votre Gmail, où l'extension peut les identifier et les organiser.

Configuration

  1. Trouvez votre email de notification dans Odoo
    Vérifiez un email de notification Odoo récent dans Gmail et regardez l'adresse de l'expéditeur (ex : notifications@votreentreprise.com). Vous pouvez aussi demander à votre administrateur Odoo l'email de notification sortant.
  2. Ouvrez les Paramètres de l'extension
    Dans la barre latérale Gmail, cliquez sur le bouton menu (trois points verticaux) et sélectionnez Paramètres.
  3. Entrez l'email de notification
    Allez dans l'onglet Notifications et collez l'adresse email. L'extension commencera à identifier les emails de notification immédiatement.
Astuce : Si vous ne savez pas quel email Odoo utilise pour les notifications, vérifiez un email de notification récent dans Gmail et regardez l'adresse de l'expéditeur.

Badges de Module

Des badges colorés apparaissent à côté des emails de notification dans votre boîte de réception, indiquant quel module Odoo a déclenché la notification :

  • Opportunité (CRM) — badge violet
  • Tâche (Projet) — badge bleu
  • Ticket (Helpdesk) — badge orange
  • Facture (Facturation) — badge vert
  • Et plus : Achats, Mailing, Feuilles de temps, etc.

Badges de Mention

Quand quelqu'un vous mentionne dans un enregistrement Odoo (avec @ ou par nom), un badge spécial apparaît sur l'email de notification :

  • Badge rouge "Mentionné": Vous avez une mention non lue — vous n'avez pas ouvert ce fil dans Gmail depuis la mention
  • Badge gris "Mentionné": Vous avez déjà vu cette mention — vous avez ouvert le fil dans Gmail après avoir été mentionné
Comment ça marche : La détection des mentions est basée sur les messages stockés dans l'enregistrement Odoo, pas sur l'email lui-même. L'extension vérifie le chatter de l'enregistrement pour les @mentions ou les références à votre nom. Cela signifie :
  • Si vous êtes récemment passé de "Gérer dans Odoo" à "Gérer par e-mails", d'anciens fils peuvent afficher un badge "Mentionné" pour des mentions survenues avant que vous ne commenciez à recevoir des emails — même si la mention n'apparaîtra dans aucun email Gmail.
  • Le statut lu/non lu est déterminé par l'ouverture ou non du fil dans Gmail après la mention. Une fois le fil ouvert, le badge devient gris.

Accès Rapide

Un raccourci "Odoo Notifications" apparaît dans la barre latérale gauche de Gmail, sous vos libellés. Cliquez dessus pour filtrer instantanément votre boîte et n'afficher que les emails de notification Odoo.

Note : Le raccourci n'apparaît qu'après avoir configuré un email de notification dans Paramètres > Notifications.

Personnalisation

Chaque fonctionnalité de notification peut être activée/désactivée indépendamment dans Paramètres > Notifications :

  • Badges de module — afficher/masquer les badges colorés par module
  • Badges de mention — afficher/masquer les badges de détection de mentions
  • Raccourci barre latérale — afficher/masquer le lien Odoo Notifications dans la barre latérale Gmail

business Support multi-sociétés Plus

Si votre utilisateur Odoo a accès à plusieurs sociétés, l'extension vous laisse choisir laquelle est active. La société active détermine quels enregistrements la barre latérale vous montre et dans quelle société chaque nouveau lead, tâche, ticket ou contact est créé.

Prérequis

  • Plan PLUS
  • Un utilisateur Odoo avec accès à deux sociétés ou plus (dans Odoo : Paramètres > Utilisateurs > Sociétés autorisées)

Le sélecteur de société

Quand vous avez accès à plus d'une société, une icône de bâtiment apparaît dans l'en-tête de la barre latérale, à côté du bouton des paramètres (⚙) :

  • Survolez l'icône pour voir le nom de la société active (« Operating in {société} »)
  • Cliquez sur l'icône pour ouvrir une liste déroulante avec toutes les sociétés auxquelles vous avez accès
  • Choisissez une société avec les boutons radio. La liste se ferme et la barre latérale se rafraîchit avec les données de la nouvelle société
  • Si vous n'avez accès qu'à une seule société, l'icône ne s'affiche pas
Persistance : Votre sélection est enregistrée par instance Odoo : en rouvrant Gmail, vous retrouvez la société dans laquelle vous travailliez. Se déconnecter efface la sélection enregistrée.

Ce que la société active affecte

Une fois une société choisie, chaque opération de la barre latérale y est limitée :

  • Enregistrements que vous voyez: Les opportunités, tâches, tickets et contacts listés dans la barre latérale sont filtrés à ceux visibles dans la société active
  • Enregistrements que vous créez: Les nouveaux leads, tâches, tickets et contacts créés depuis la barre latérale sont affectés à la société active
  • Emails enregistrés automatiquement: Les emails traités par l'enregistrement automatique sont publiés dans la société propriétaire de l'enregistrement lié (voir la note sur l'enregistrement automatique ci-dessous)

Recherche de contacts entre sociétés

Si vous ouvrez un email de quelqu'un qui n'est pas dans la société active mais existe ailleurs dans votre instance Odoo, la barre latérale vous le signale et vous propose des actions au lieu de vous laisser un état vide.

Le contact existe dans une autre société

La barre latérale montre le contact trouvé, la société où il se trouve, et deux actions :

  • Rendre visible pour toutes les sociétés: Supprime la restriction de société du contact pour que les utilisateurs de chaque société à laquelle vous avez accès puissent le voir. Demande confirmation avant d'enregistrer
  • Créer un nouveau dans cette société: Crée un contact distinct dans la société active, sans toucher à l'original

Seule la société mère existe dans une autre société

Le contact est absent, mais une société liée (trouvée par le domaine de l'email) existe dans une autre de vos sociétés. Deux actions :

  • Partager la société entre toutes les sociétés: Rend la société mère visible pour toutes les sociétés auxquelles vous avez accès, puis vous laisse créer le contact sous elle
  • Créer la société ici: Crée un enregistrement de société indépendant dans la société active et le contact sous lui
Important : Partager un contact ou une société entre toutes vos sociétés change sa visibilité pour tous les utilisateurs ayant accès à ces sociétés. L'extension demande confirmation avant d'enregistrer.

Enregistrement automatique entre sociétés

Quand vous activez l'enregistrement automatique sur un enregistrement, l'extension mémorise la société à laquelle il appartient. Les nouveaux emails de ce fil sont toujours enregistrés dans la bonne société, même si vous avez changé de société active dans la barre latérale entre-temps.

Comportement avec le plan GRATUIT

Si vous avez accès à plusieurs sociétés avec le plan GRATUIT, l'extension utilise votre société par défaut dans Odoo pour toutes les opérations. L'icône du sélecteur n'apparaît pas et la recherche entre sociétés n'est pas proposée. Passer à PLUS débloque les deux.

Limites connues

  • Non synchronisé avec le sélecteur Odoo du navigateur: L'extension garde sa propre société active et l'envoie avec chaque requête, tandis que le sélecteur de société d'Odoo utilise la session de votre navigateur. Les deux sont indépendants — changer de société dans votre onglet Odoo ne change pas la société active dans Gmail, et inversement.
  • Société par défaut modifiée dans Odoo: Si votre administrateur change votre société par défaut dans Odoo, l'extension continue d'utiliser la valeur en cache jusqu'à ce que vous vous déconnectiez et reconnectiez.
  • Une société à la fois: Le sélecteur est à choix unique : la barre latérale montre les enregistrements d'une société à la fois. Pour voir ceux d'une autre société, basculez vers elle.
  • Erreur d'accès après la création d'un enregistrement: Quand vous créez une opportunité, une tâche, un ticket ou un contact depuis la barre latérale (ou cliquez sur « Voir dans Odoo »), l'extension ouvre l'enregistrement dans un nouvel onglet. Odoo le charge sous la société actuellement active dans votre session Odoo du navigateur (la session par cookie — voir ci-dessus). Si cette société diffère de celle où l'enregistrement a été créé, Odoo peut afficher une erreur d'accès. Avec le plan GRATUIT, l'extension crée toujours les enregistrements dans votre société Odoo par défaut — pour les voir, basculez votre onglet Odoo vers cette même société. Avec le plan PLUS, gardez la même société sélectionnée dans le sélecteur de la barre latérale et dans votre onglet Odoo. Odoo 18 et versions ultérieures changent automatiquement votre société active à l'ouverture d'un enregistrement appartenant à une autre de vos sociétés autorisées ; le problème se voit donc surtout sur Odoo 17 et antérieurs.

help Dépannage

L'extension n'apparaît pas dans Gmail

  1. Vérifiez que l'extension est activée: Allez sur chrome://extensions/ et assurez-vous que le commutateur est sur ON
  2. Actualisez Gmail: Appuyez sur Ctrl+Shift+R (Windows/Linux) ou Cmd+Shift+R (Mac)
  3. Rechargez l'extension: Cliquez sur "Recharger" sur la carte de l'extension

Message "Contact non trouvé"

  • Email non présent dans Odoo: Cliquez sur "Créer un contact" pour l'ajouter
  • Différence d'orthographe de l'email: Mettez à jour l'email dans Odoo ou créez un nouveau contact

La barre latérale est vide ou rien ne se charge après la connexion

C'est presque toujours la session de connexion, pas une clé API. Vérifiez, dans l'ordre :

  1. Êtes-vous connecté à Odoo dans le même navigateur ?
    Ouvrez votre URL Odoo dans un autre onglet — si une connexion vous est demandée, connectez-vous, puis rechargez Gmail. L'extension utilise cette session.
  2. L'URL est-elle exacte ?
    Utilisez https://, l'URL réelle de votre instance, et sans barre oblique finale. Auto-hébergé : utilisez votre domaine personnalisé.
  3. Ouvrez un véritable email, pas la liste de la boîte de réception
    La fiche contact ne se charge que dans un email ouvert. Rechargez ensuite Gmail (Cmd/Ctrl+Shift+R).

Toujours bloqué ? Répondez à votre email de bienvenue ou envoyez-nous une capture d'écran — nous répondons vite : odoointegrations.team@gmail.com

« Session expirée » ou reconnexion demandée

Votre session Odoo a expiré. Ouvrez simplement Odoo, reconnectez-vous, et l'extension se reconnecte — rien d'autre à gérer.

Une clé API facultative peut être ajoutée en secours pour que les fonctionnalités en arrière-plan continuent de fonctionner quand la session expire : Paramètres → Compte → Connexion.

Le popup de permissions n'apparaît pas (Odoo auto-hébergé)

Lors de la connexion à une instance Odoo auto-hébergée (URL personnalisée), Chrome nécessite des permissions supplémentaires. Si le popup de permissions n'apparaît pas :

  1. Permission requise: Les URLs Odoo personnalisées (autres que *.odoo.com) nécessitent une permission Chrome supplémentaire pour communiquer avec votre serveur
  2. D'autres extensions peuvent bloquer les popups: Les bloqueurs de publicités, extensions de confidentialité ou bloqueurs de popups peuvent empêcher l'affichage du dialogue de permissions Chrome
  3. Solution: Désactivez temporairement les autres extensions (notamment les bloqueurs de publicités) et réessayez de vous connecter. Réactivez-les après avoir accordé la permission
Note : Cette permission n'est requise qu'une seule fois par URL Odoo. Après l'avoir accordée, l'extension fonctionnera normalement avec votre instance auto-hébergée.

quiz Questions fréquentes

Général

Q : Quelles versions d'Odoo sont supportées ?

Odoo 14.0 et versions supérieures sont supportées. Compatible avec les installations on-premise (Community/Enterprise), Odoo.sh et Odoo Online (plan Custom uniquement, car les plans Standard et Free n'ont pas accès à l'API externe).

Q : Ça fonctionne avec d'autres navigateurs ?

Actuellement, l'extension ne supporte que Google Chrome. Le support d'autres navigateurs pourrait être ajouté à l'avenir.

Q : Ai-je besoin d'une clé API ?

Non. Seulement votre URL Odoo et une session Odoo active dans le même navigateur. Une clé API est facultative — uniquement pour l'enregistrement automatique en arrière-plan (PLUS) ou comme solution de secours si votre session expire.

Q : Mes données sont-elles sécurisées ?

Oui. L'extension n'accède qu'aux emails que vous consultez activement et communique de manière sécurisée avec votre instance Odoo via HTTPS, en utilisant votre propre session Odoo. Si vous ajoutez une clé API facultative, elle est stockée dans le stockage sécurisé de Chrome.

Abonnement

Q : Quelle est la différence entre GRATUIT et PLUS ?

GRATUIT inclut les fonctionnalités de base (consultation des contacts, enregistrement d'emails, enregistrements limités). PLUS ajoute la recherche avancée, les aperçus financiers, les contacts associés, l'organisation des emails de notification et les options de personnalisation.

Q : Puis-je annuler mon abonnement PLUS à tout moment ?

Oui, vous pouvez annuler à tout moment sans pénalité. Votre abonnement restera actif jusqu'à la fin de votre période de facturation.

Confidentialité

Q : L'extension lit-elle tous mes emails ?

Non, l'extension n'accède qu'aux emails que vous consultez activement. Elle ne scanne pas toute votre boîte mail et ne lit pas les emails en arrière-plan.

Q : Ma clé API est-elle stockée de manière sécurisée ?

Une clé API est facultative — la plupart des utilisateurs se connectent avec leur session Odoo et n'en créent jamais. Si vous en ajoutez une, elle est stockée dans le stockage sécurisé de Chrome (chrome.storage.sync), que Chrome chiffre avec les identifiants de votre compte utilisateur OS.
