Avvio rapido (2 minuti, senza chiave API)
  1. Installa l'estensione e apri Gmail.
  2. Accedi al tuo Odoo nello stesso browser Chrome — l'estensione usa la tua sessione attiva.
  3. Inserisci il tuo URL di Odoo (es. https://tuaazienda.odoo.com — HTTPS, senza barra finale) e clicca Connetti.
Apri una qualsiasi email e il tuo contatto Odoo appare nella barra laterale. Tutto qui — niente chiave API, niente codice.

download Installazione

Prerequisiti

  • Browser Chrome: Versione 88 o superiore
  • Istanza Odoo: Versione 14.0 o superiore (on-premise, Odoo.sh o Odoo Online piano Custom)

Installazione Passo per Passo

  1. Visita Chrome Web Store
    Vai a Integrazione Odoo per Gmail o cerca "Odoo Integration for Gmail" nel Chrome Web Store.
  2. Aggiungi a Chrome
    Clicca il pulsante "Aggiungi a Chrome" e rivedi i permessi richiesti.
  3. Verifica Installazione
    Vedrai un messaggio di conferma e l'icona dell'estensione apparirà nella barra degli strumenti di Chrome.
  4. Fissa l'Estensione (Consigliato)
    Clicca l'icona del puzzle nella barra degli strumenti di Chrome, trova "Odoo Integration for Gmail" e clicca l'icona della puntina.
Nota su "Navigazione sicura avanzata": Se vedi un messaggio che dice "Questa estensione non è considerata attendibile da Navigazione sicura avanzata", questo è normale per le estensioni appena pubblicate. L'estensione è stata rivista e approvata dal Chrome Web Store.

settings Configurazione

Configurazione Iniziale

  1. Apri Gmail
    Vai a mail.google.com e apri qualsiasi email per vedere la barra laterale.
  2. Accedi a Odoo
    Assicurati di aver effettuato l'accesso alla tua istanza Odoo nello stesso browser — l'estensione si connette usando la tua sessione attiva.
  3. Inserisci il tuo URL Odoo
    Inserisci l'URL della tua istanza Odoo (es. https://tuaazienda.odoo.com). Usa HTTPS e non includere una barra finale.
  4. Connetti
    Clicca Connetti e l'estensione verificherà la tua sessione attiva. Una chiave API è facoltativa — necessaria solo per la registrazione automatica in background, o come riserva se la tua sessione scade.
info Aggiungere una chiave API è facoltativo. Se ne aggiungi una, i passaggi per crearla variano tra le versioni di Odoo. Consulta la guida dettagliata per la tua versione →

Aggiornamento Configurazione

Per cambiare il tuo URL Odoo, esci e riconnettiti:

  1. Clicca il pulsante menu (tre punti verticali ⋮) nella barra laterale
  2. Seleziona Esci
  3. Inserisci i tuoi nuovi dati

Per aggiungere o aggiornare la chiave API in seguito, apri Impostazioni → Account → Connessione — senza bisogno di uscire.

mail Logging Email

Come Registrare un'Email

  1. Apri l'email che vuoi registrare
  2. Visualizza il contatto nella barra laterale
  3. Trova il record in cui vuoi registrare (opportunità, attività o ticket)
  4. Clicca l'icona della busta accanto al record
  5. Il pulsante diventa un segno di spunta verde quando registrato

Cosa Viene Registrato

  • Oggetto email (nell'intestazione)
  • Corpo email (formattato HTML)
  • Informazioni mittente/destinatario
  • Data e ora
  • Tutti gli allegati (file, immagini, PDF)
  • Footer "Registrato da Gmail Inbox"
Registrato come Nota Interna: Le email vengono registrate come note interne nel chatter di Odoo. Questo significa che solo gli utenti interni (dipendenti) possono vederle. Gli utenti del portale (clienti) NON vedranno le email registrate. Le notifiche vengono inviate solo ai follower interni del record.

Rilevamento Duplicati

L'estensione ha due livelli di rilevamento duplicati:

  1. Indicatore visivo (0-6 ore): Ti aiuta a identificare a colpo d'occhio quali record hanno già questa email
  2. Validazione lato server (sempre): Odoo previene i duplicati anche se l'indicatore visivo è scaduto
Importante: Anche senza l'indicatore visivo, non puoi creare accidentalmente duplicati. Il server di Odoo valida ogni richiesta di logging e rifiuta i duplicati.

sync Auto-logging Plus

Auto-logging links a Gmail thread to one or more Odoo records so that every new email in that thread is logged into the record's chatter automatically — no need to click the envelope icon for each reply.

Private Beta: Auto-logging is currently in private beta. It will be available to all PLUS subscribers soon. To request early access, contact support.

Requirements

  • PLUS plan (and beta access during private beta)
  • Gmail API connection (granted from Settings > Auto-logging > Connect)
  • Auto-logging toggle enabled in Settings > Auto-logging

How to Activate Auto-logging on a Thread

  1. Open the email thread in Gmail and view the contact in the sidebar
  2. Find the record you want to track (opportunity, task, ticket, or contact)
  3. Click the sync icon next to the record. The icon turns green when active and a confirmation toast appears
  4. From now on, every new email in this thread is logged automatically into the chatter of that record
Only future emails: Auto-logging is purely forward-looking. Emails that already existed in the thread before activation are NOT logged retroactively. To log older emails, use the manual envelope button as usual.

What Gets Logged

Each new email is added as an internal note in the record's chatter, with:

  • A Gmail-style header (From, To, Date, Subject)
  • The email body without quoted history
  • The original date of the email (not the time it was processed)
  • Linked partners — recipients are matched in Odoo by email and attached to the message
  • An "Auto-logged from Gmail" footer linking back to the extension
Logged as Internal Note: Auto-logged emails are saved as internal notes (mt_note), not public messages. Only internal Odoo users can see them — portal users (customers) won't.

Linking a Thread to Several Records

You can activate auto-logging for the same thread on multiple records (for example, an opportunity and a task related to the same conversation):

  • 1 thread → 1 record: standard case.
  • 1 thread → several records: every new email is logged on each linked record sequentially.
  • The same record cannot be linked twice from the same thread — duplicates are rejected.

Managing Active Auto-logs

From the sidebar (per thread)

When the open thread has any active mappings, an "Auto-logging (N)" section appears at the top of the sidebar:

  • Click the section to expand it and see every record linked to the current thread
  • Click on a record's name to open it in Odoo
  • Click the ✕ button to deactivate auto-logging for that specific record

From Settings (across all threads)

Open Settings > Auto-logging to manage every active mapping:

  • Enable or disable auto-logging globally with a single toggle
  • Connect or reconnect the Gmail API
  • See the list of active mappings, the count, and the time of the last successful check
  • Deactivate all mappings at once with the "Deactivate all" button

Limits & Edge Cases

  • Not retroactive: Only emails that arrive after activation are logged. Existing emails must be logged manually.
  • Auto-cleanup after 90 days: A mapping with no new email activity for 90 days is removed automatically. Activate it again to resume tracking.
  • Record deleted in Odoo: If the linked record is deleted in Odoo, the mapping is removed automatically the next time the extension tries to log into it.
  • Gmail disconnected: If the Gmail API connection expires or is revoked, auto-logging stops for all threads until you reconnect from Settings > Auto-logging.

groups Lavorare con i contatti correlati Plus

La sezione Contatti correlati nella barra laterale elenca i colleghi della stessa azienda del mittente dell'email. Da lì puoi eseguire azioni rapide su ogni contatto senza uscire da Gmail.

Aprire un contatto in Odoo

Clicca in qualsiasi punto della scheda del contatto — l'avatar, il nome o la mansione — per aprire la sua scheda in Odoo in una nuova scheda.

Inviare una nuova email a un contatto

Clicca sull'icona dell'email accanto all'indirizzo del contatto. Se non hai nessuna bozza aperta, Gmail apre una nuova finestra di composizione con quell'indirizzo precompilato nel campo A.

Aggiungere un contatto a un'email che stai già scrivendo

Se hai già aperta una finestra di composizione di Gmail quando clicchi sull'icona dell'email, il contatto viene aggiunto come destinatario di quella bozza invece di aprirne una nuova. L'estensione decide dove: se il campo A è vuoto, il contatto va lì; se contiene già qualcuno, il contatto viene aggiunto in CC e la finestra di composizione viene ridotta a icona così puoi continuare a cliccare su altri contatti per costruire la lista dei destinatari.

Copiare l'email o il telefono di un contatto

Clicca sul testo dell'email o del telefono — non sull'icona accanto — per copiarlo negli appunti. Una piccola notifica conferma la copia.

Chiamare un contatto

Clicca sull'icona del telefono per avviare una chiamata. L'azione viene delegata all'app che il tuo sistema ha associato ai numeri di telefono — il dialer del tuo cellulare, FaceTime, Skype, Microsoft Teams, ecc. Se non hai nessuna app configurata, il clic non fa nulla.

history Navigare nelle attività recenti Plus

La sezione Attività recenti mostra gli ultimi messaggi e conversazioni dalla scheda del contatto in Odoo, in modo che tu possa vedere cosa è stato detto senza uscire da Gmail.

Vedere esattamente quando è avvenuta un'attività

Ogni voce mostra una data relativa come "5 giorni fa". Passa il mouse sulla data per vedere il giorno e l'ora esatti, formattati secondo la lingua del tuo browser.

Saltare a un messaggio specifico in Odoo

Clicca su una voce delle Attività recenti per aprire la scheda Odoo correlata in una nuova scheda del browser. Su Odoo 18 e successivi, il clic va un passo oltre: Odoo scorre fino a quel messaggio esatto e lo evidenzia. Su Odoo 16 e 17 la scheda si apre normalmente — il clic ti risparmia comunque la navigazione, ma dovrai scorrere tu per trovare il messaggio.

notifications Organizzazione Notifiche Plus

Organizza le email di notifica Odoo direttamente nella tua casella di posta Gmail. L'estensione identifica le email di notifica da Odoo e aggiunge badge visivi per vedere istantaneamente a quale modulo appartengono e se sei stato menzionato.

Prerequisito: Attivare le Notifiche via Email in Odoo

Affinché questa funzione funzioni, il tuo utente Odoo deve essere configurato per ricevere notifiche via email (non nella posta di Odoo):

  1. In Odoo, vai al tuo Profilo (menu in alto a destra > Il Mio Profilo)
  2. In Preferenze, trova l'impostazione "Notifica"
  3. Seleziona "Gestisci via email" (invece di "Gestisci in Odoo")
Questo assicura che Odoo invii le email di notifica al tuo Gmail, dove l'estensione può identificarle e organizzarle.

Configurazione

  1. Trova la tua email di notifica in Odoo
    Controlla un'email di notifica Odoo recente in Gmail e guarda l'indirizzo del mittente (es: notifications@tuaazienda.com). In alternativa, chiedi al tuo amministratore Odoo l'email di notifica in uscita.
  2. Apri le Impostazioni dell'estensione
    Nella barra laterale di Gmail, clicca il pulsante menu (tre punti verticali) e seleziona Impostazioni.
  3. Inserisci l'email di notifica
    Vai alla scheda Notifiche e incolla l'indirizzo email. L'estensione inizierà a identificare le email di notifica immediatamente.
Suggerimento: Se non sei sicuro di quale email Odoo usa per le notifiche, controlla un'email di notifica recente in Gmail e guarda l'indirizzo del mittente.

Badge dei Moduli

Badge colorati appaiono accanto alle email di notifica nella tua casella di posta, mostrando quale modulo Odoo ha generato la notifica:

  • Opportunità (CRM) — badge viola
  • Attività (Progetto) — badge blu
  • Ticket (Helpdesk) — badge arancione
  • Fattura (Fatturazione) — badge verde
  • E altro: Acquisti, Mailing, Fogli ore, ecc.

Badge delle Menzioni

Quando qualcuno ti menziona in un record Odoo (usando @ o per nome), un badge speciale appare sull'email di notifica:

  • Badge rosso "Menzionato": Hai una menzione non letta — non hai aperto questo thread in Gmail dalla menzione
  • Badge grigio "Menzionato": Hai già visto questa menzione — hai aperto il thread in Gmail dopo essere stato menzionato
Come funziona: Il rilevamento delle menzioni si basa sui messaggi memorizzati nel record Odoo, non sull'email stessa. L'estensione controlla il chatter del record cercando @menzioni o riferimenti al tuo nome. Questo significa:
  • Se sei passato di recente da "Gestisci in Odoo" a "Gestisci via email", thread più vecchi possono mostrare un badge "Menzionato" per menzioni avvenute prima di iniziare a ricevere email — anche se la menzione non apparirà in nessuna email Gmail.
  • Lo stato letto/non letto è determinato dall'apertura o meno del thread in Gmail dopo la menzione. Una volta aperto il thread, il badge diventa grigio.

Accesso Rapido

Un collegamento "Odoo Notifications" appare nella barra laterale sinistra di Gmail, sotto le tue etichette. Cliccalo per filtrare istantaneamente la tua casella e mostrare solo le email di notifica Odoo.

Nota: Il collegamento appare solo dopo aver configurato un'email di notifica in Impostazioni > Notifiche.

Personalizzazione

Ogni funzione di notifica può essere attivata/disattivata indipendentemente in Impostazioni > Notifiche:

  • Badge dei moduli — mostra/nascondi i badge colorati per modulo
  • Badge delle menzioni — mostra/nascondi i badge di rilevamento menzioni
  • Collegamento barra laterale — mostra/nascondi il link Odoo Notifications nella barra laterale di Gmail

business Supporto multi-azienda Plus

Se il tuo utente Odoo ha accesso a più aziende, l'estensione ti permette di scegliere quale è attiva. L'azienda attiva decide quali record ti mostra la barra laterale e in quale azienda viene creato ogni nuovo lead, attività, ticket o contatto.

Requisiti

  • Piano PLUS
  • Un utente Odoo con accesso a due o più aziende (in Odoo: Impostazioni > Utenti > Aziende consentite)

Il selettore di azienda

Quando hai accesso a più di un'azienda, un'icona a forma di edificio appare nell'intestazione della barra laterale, accanto al pulsante delle impostazioni (⚙):

  • Passa il mouse sull'icona per vedere il nome dell'azienda attiva ("Operating in {azienda}")
  • Clicca l'icona per aprire un menu a tendina con tutte le aziende a cui hai accesso
  • Scegli un'azienda con i pulsanti radio. Il menu si chiude e la barra laterale si aggiorna con i dati della nuova azienda
  • Se hai accesso a una sola azienda, l'icona non viene mostrata
Persistenza: La tua selezione viene salvata per istanza Odoo, quindi riaprendo Gmail ritrovi l'azienda in cui stavi lavorando. Uscire cancella la selezione salvata.

Cosa influenza l'azienda attiva

Una volta scelta un'azienda, ogni operazione nella barra laterale è limitata ad essa:

  • Record che vedi: Opportunità, attività, ticket e contatti elencati nella barra laterale sono filtrati a quelli visibili nell'azienda attiva
  • Record che crei: Nuovi lead, attività, ticket e contatti creati dalla barra laterale vengono assegnati all'azienda attiva
  • Email registrate automaticamente: Le email elaborate dalla registrazione automatica vengono pubblicate nell'azienda a cui appartiene il record collegato (vedi la nota sulla registrazione automatica qui sotto)

Ricerca contatti cross-azienda

Se apri un'email di qualcuno che non è nell'azienda attiva ma esiste altrove nella tua istanza Odoo, la barra laterale te lo segnala e ti offre azioni invece di lasciarti uno stato vuoto.

Il contatto esiste in un'altra azienda

La barra laterale mostra il contatto trovato, l'azienda in cui si trova e due azioni:

  • Rendi visibile a tutte le aziende: Rimuove la restrizione di azienda del contatto così gli utenti di ogni azienda a cui hai accesso possono vederlo. Chiede conferma prima di salvare
  • Crea nuovo in questa azienda: Crea un record contatto separato nell'azienda attiva, lasciando intatto l'originale

Solo l'azienda padre esiste in un'altra azienda

Il contatto manca, ma un'azienda correlata (trovata tramite il dominio email) esiste in un'altra delle tue aziende. Due azioni:

  • Condividi l'azienda tra tutte le aziende: Rende l'azienda padre visibile a tutte le aziende a cui hai accesso, poi ti permette di creare il contatto al suo interno
  • Crea l'azienda qui: Crea un record azienda indipendente nell'azienda attiva e il contatto al suo interno
Importante: Condividere un contatto o un'azienda tra tutte le tue aziende ne cambia la visibilità per ogni utente con accesso a quelle aziende. L'estensione chiede conferma prima di salvare.

Registrazione automatica tra aziende

Quando attivi la registrazione automatica su un record, l'estensione ricorda a quale azienda appartiene quel record. Le nuove email di quel thread vengono sempre registrate nell'azienda giusta, anche se nel frattempo hai cambiato azienda attiva nella barra laterale.

Comportamento nel piano FREE

Se hai accesso a più aziende nel piano FREE, l'estensione usa la tua azienda predefinita in Odoo per ogni operazione. L'icona del selettore non appare e le ricerche cross-azienda non vengono offerte. Passare a PLUS sblocca entrambe.

Limiti noti

  • Non sincronizzato con il selettore Odoo del browser: L'estensione mantiene la propria azienda attiva e la invia con ogni richiesta, mentre il selettore di azienda di Odoo usa la sessione del tuo browser. I due sono indipendenti — cambiare azienda nella tua scheda Odoo non cambia l'azienda attiva in Gmail e viceversa.
  • Azienda predefinita cambiata in Odoo: Se il tuo amministratore cambia la tua azienda predefinita in Odoo, l'estensione continua a usare il valore in cache finché non esci e rientri.
  • Un'azienda alla volta: Il selettore è a scelta singola: la barra laterale mostra i record di un'azienda alla volta. Per vedere i record di un'altra azienda, passa ad essa.
  • Errore di accesso dopo la creazione di un record: Quando crei un'opportunità, attività, ticket o contatto dalla barra laterale (o clicchi "Vedi in Odoo" su uno di essi), l'estensione apre il record in una nuova scheda del browser. Odoo lo carica sotto l'azienda attualmente attiva nella tua sessione Odoo del browser (la sessione cookie — vedi sopra). Se quell'azienda è diversa da quella in cui il record è stato creato, Odoo può mostrare un errore di accesso. Nel piano FREE, l'estensione crea sempre i record nella tua azienda predefinita di Odoo — per vederli, porta la scheda Odoo del browser su quella stessa azienda predefinita. Nel piano PLUS, mantieni la stessa azienda selezionata sia nel selettore della barra laterale sia nella tua scheda Odoo. Odoo 18 e successivi cambiano automaticamente l'azienda attiva quando apri un record che appartiene a un'altra delle tue aziende consentite, quindi il problema è visibile soprattutto su Odoo 17 e precedenti.

help Risoluzione Problemi

L'Estensione Non Appare in Gmail

  1. Verifica che l'estensione sia abilitata: Vai a chrome://extensions/ e assicurati che l'interruttore sia ON
  2. Aggiorna Gmail: Premi Ctrl+Shift+R (Windows/Linux) o Cmd+Shift+R (Mac)
  3. Ricarica l'estensione: Clicca "Ricarica" sulla scheda dell'estensione

Messaggio "Contatto Non Trovato"

  • Email non in Odoo: Clicca "Crea Contatto" per aggiungerlo
  • Errore ortografico email: Aggiorna l'email in Odoo o crea un nuovo contatto

La barra laterale è vuota o non si carica nulla dopo la connessione

È quasi sempre la sessione di accesso, non una chiave API. Controlla, in ordine:

  1. Hai effettuato l'accesso a Odoo nello stesso browser?
    Apri il tuo URL di Odoo in un'altra scheda — se ti chiede di accedere, accedi e poi ricarica Gmail. L'estensione usa quella sessione.
  2. L'URL è esatto?
    Usa https://, l'URL reale della tua istanza e senza barra finale. Self-hosted: usa il tuo dominio personalizzato.
  3. Apri un'email vera, non l'elenco della posta in arrivo
    La scheda del contatto si carica solo dentro un'email aperta. Poi ricarica Gmail (Cmd/Ctrl+Shift+R).

Ancora bloccato? Rispondi alla tua email di benvenuto o inviaci uno screenshot — rispondiamo in fretta: odoointegrations.team@gmail.com

“Sessione scaduta” o ti viene chiesto di riconnetterti

La tua sessione Odoo è scaduta. Apri semplicemente Odoo, accedi di nuovo e l'estensione si riconnette — nient'altro da gestire.

Una chiave API facoltativa può essere aggiunta come riserva, così le funzioni in background continuano a funzionare quando la sessione scade: Impostazioni → Account → Connessione.

Il popup dei permessi non appare (Odoo Self-Hosted)

Quando ci si connette a un'istanza Odoo self-hosted (URL personalizzato), Chrome richiede permessi aggiuntivi. Se il popup dei permessi non appare:

  1. Permesso richiesto: Gli URL Odoo personalizzati (non *.odoo.com) richiedono un permesso Chrome aggiuntivo per comunicare con il tuo server
  2. Altre estensioni possono bloccare i popup: Gli ad blocker, le estensioni per la privacy o i blocca popup possono impedire la visualizzazione della finestra di dialogo dei permessi di Chrome
  3. Soluzione: Disabilita temporaneamente altre estensioni (specialmente gli ad blocker) e riprova a connetterti. Riabilitale dopo aver concesso il permesso
Nota: Questo permesso è richiesto solo una volta per URL Odoo. Dopo averlo concesso, l'estensione funzionerà normalmente con la tua istanza self-hosted.

quiz Domande Frequenti

Generale

D: Quali versioni di Odoo sono supportate?

Sono supportate Odoo 14.0 e superiori. Compatibile con installazioni on-premise (Community/Enterprise), Odoo.sh e Odoo Online (solo piano Custom, poiché i piani Standard e Free non hanno accesso all'API esterna).

D: Funziona con altri browser?

Attualmente, l'estensione supporta solo Google Chrome. Il supporto per altri browser potrebbe essere aggiunto in futuro.

D: Ho bisogno di una chiave API?

No. Solo il tuo URL di Odoo e una sessione Odoo attiva nello stesso browser. Una chiave API è facoltativa — solo per la registrazione automatica in background (PLUS) o come riserva se la tua sessione scade.

D: I miei dati sono sicuri?

Sì. L'estensione accede solo alle email che visualizzi attivamente e comunica in modo sicuro con la tua istanza Odoo tramite HTTPS, usando la tua stessa sessione Odoo. Se aggiungi una chiave API facoltativa, viene memorizzata nello storage sicuro di Chrome.

Abbonamento

D: Qual è la differenza tra FREE e PLUS?

FREE include le funzionalità base (visualizzazione contatti, logging email, record limitati). PLUS aggiunge ricerca avanzata, approfondimenti finanziari, contatti correlati, organizzazione delle email di notifica e opzioni di personalizzazione.

D: Posso annullare il mio abbonamento PLUS in qualsiasi momento?

Sì, puoi annullare in qualsiasi momento senza penali. Il tuo abbonamento rimarrà attivo fino alla fine del periodo di fatturazione corrente.

Privacy

D: L'estensione legge tutte le mie email?

No, l'estensione accede solo alle email che visualizzi attivamente. Non scansiona l'intera casella di posta né legge le email in background.

D: La mia chiave API è memorizzata in modo sicuro?

Una chiave API è facoltativa — la maggior parte degli utenti si connette con la propria sessione Odoo e non ne crea mai una. Se ne aggiungi una, viene memorizzata nello storage sicuro di Chrome (chrome.storage.sync), che Chrome crittografa usando le credenziali del tuo account utente del sistema operativo.
